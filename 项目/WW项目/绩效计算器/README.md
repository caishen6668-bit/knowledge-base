# WW 项目绩效计算器

## 项目简介

WW 项目的每日绩效计算工具。读取每日员工日报 Excel，匹配目标表，按催回率公式计算每人每天的绩效系数和绩效金额，导出汇总报告。

核心设计目标：新增目标版本、新增阶段、人员变动、命名变动都**不需要改代码**，只修改 `config.py`。

## 项目结构

```
绩效计算器/
├── README.md               ← 本文件
├── WW项目绩效计算器.py       ← 主入口 / CLI / 流程编排
├── config.py               ← 全部可变规则（唯一需要日常维护的文件）
├── ARCHITECTURE.md          ← 架构设计文档
├── WW项目绩效汇总.xlsx       ← 输出示例（不上传 GitHub）
└── 日报/                    ← 每日日报输入（不上传 GitHub）
    ├── 10号.xlsx
    ├── 11号.xlsx
    ├── 12号.xlsx
    ├── 13号.xlsx
    └── 14号.xlsx
```

> ⚠️ 日报目录和绩效汇总 Excel 包含真实业务数据，已在 `.gitignore` 中排除，不上传 GitHub。

## 运行方式

```bash
python WW项目绩效计算器.py                     # 处理全部日报
python WW项目绩效计算器.py --days 10-14        # 指定日期范围
python WW项目绩效计算器.py --month 2026-07     # 手动指定年月
python WW项目绩效计算器.py --check             # 只读取+校验，不出报表
```

## 输入

| 输入 | 来源 | 说明 |
|------|------|------|
| 日报 Excel | `../日报/N号.xlsx` | 员工每日数据（分案/应催/结清/回款等 9 个维度） |
| 目标表 Excel | 配置中的路径 | 多版本绩效目标（按生效日期索引） |

## 输出

- `{起}-{止}号WW项目绩效汇总.xlsx` — 每日 sheet + 绩效汇总 sheet
- 控制台警告汇总（结构校验 + 数据校验）

## 注意事项

- **规则变更只改 config.py，不动算法代码**
- 跨表匹配一律用**工号**（feimi004~012），姓名仅展示
- 日报表头按文字定位列，列顺序变化不影响读取
- 队列→阶段映射未列出的队列名 = 阶段名本身
- 数据日期自动推断（从文件名 + sheet 名），支持 `--month` 手动覆盖
- 缺勤日输出行为可配置（填 0 / 留空）
