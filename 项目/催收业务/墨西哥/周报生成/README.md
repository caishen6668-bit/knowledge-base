# 周报生成

## 项目简介

每周从 Quick BI 获取业绩、出勤、催收数据，按照周报模板格式生成周度员工排名和阶段达成 Excel 报告。

## 项目结构

```
周报生成/
├── README.md              ← 本文件
└── 周度员工排行及阶段.py    ← 主脚本
```

## 运行方式

```bash
python 周度员工排行及阶段.py
```

> 参数（日期范围、模板路径、输出路径、CPP 值）硬编码在脚本顶部，按需修改。

## 输入

| 输入 | 来源 | 说明 |
|------|------|------|
| 案件量数据 | Quick BI API `c2f93e0fa45b` | 到期案件（按包体/新老客） |
| 出勤数据 | Quick BI API `7f9969dc9020` | 排班记录 |
| 业绩数据 | Quick BI API `c4f429db60b3` | 催收员业绩 |
| 回收率数据 | Quick BI API `524c3ccd429c` | 首逾/回收率 |
| 周报模板 | 本地 Excel | 格式 + 公式模板 |

## 输出

- 周报 Excel 文件（含多个 Sheet：整体、新客、老客、人数预估等）
- 输出到桌面（路径在脚本顶部 `OUT` 变量）

## 依赖

```bash
pip install requests pandas openpyxl
```

Quick BI HMAC-SHA1 签名直接在脚本内实现。

## 注意事项

- 每次运行需手动更新脚本顶部的日期范围（`DS`/`DE`/`PERIOD`/`TODAY`）和模板路径
- 模版内公式和格式较复杂，修改时注意保留
