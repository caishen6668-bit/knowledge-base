# 通用工具

## 项目简介

催收业务的一次性工具脚本集合，按需手动运行，不参与自动化流程。

## 工具清单

| 脚本 | 用途 | 输入 | 输出 |
|------|------|------|------|
| `分案派单.py` | 从周报解析员工数据，按阶段和组别分案派单 | 周报 Excel | 派单结果 |
| `同步报表.py` | 从周报 K-M 列提取员工→阶段映射，同步到其他 Sheet | 周报 Excel | 更新后的周报 |
| `写入分案.py` | 将分案结果写回周报的人数预估 Sheet（D0/D1 人均持案 → 35） | 周报 Excel | 更新后的周报 |
| `更新人力.py` | 从 Quick BI 获取最新人力数据并更新 Excel | Quick BI API | 更新后的 Excel |
| `校验人力.py` | 计算未来 7 天日均到期量（分期/单期×新客/老客） | 硬编码数值 | 控制台输出 |
| `Excel审核.py` | 审核日报 Excel 的行结构和数据完整性 | 日报 Excel | 控制台检查报告 |
| `修复定稿.py` | 修复周报 Excel 格式（颜色/边框/对齐/字体） | 周报 Excel | 美化后的周报 |
| `美化格式.py` | 统一修复周报 Excel 的格式细节（边框/字体/对齐） | 周报 Excel | 美化后的周报 |

## 运行方式

每个脚本独立运行，路径通常硬编码在脚本顶部，使用前需手动修改为实际路径。

```bash
python 分案派单.py
python 同步报表.py
# ... 以此类推
```

## 依赖

```bash
pip install openpyxl requests pandas
```

部分脚本需要 `alibabacloud_quickbi_public20220101` SDK 访问 Quick BI。

## 注意事项

- 这些是**一次性工具**，不是长期维护的产品
- 每次使用前检查脚本顶部的硬编码路径是否符合当前环境
- 如某工具被频繁使用，建议升级为正式项目
- 大部分脚本依赖本地 Excel 文件，文件不存在时会报错
